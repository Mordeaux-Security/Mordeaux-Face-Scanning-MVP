## 1. User Journeys

### Journey A: Open Page by searchId

**Scenario**: Developer has a `searchId` from API logs/audit and wants to view the search results.

**Flow**:
```
1. Developer opens URL: /search?id={searchId}
   â†“
2. Page loads â†’ Shows loading state
   â†“
3. API call: GET /api/v1/search/{searchId}/results
   â†“
4. [Success Path]
   - Query image displayed in query panel (top)
   - Search results displayed in match grid (center)
   - Filters and pagination controls visible (sidebar/top)
   - Debugging info available (expandable panel, bottom)
   
   [Error Path - searchId not found]
   - Show error state: "Search not found"
   - Offer: "Upload new image to search"
   
   [Error Path - searchId expired]
   - Show error state: "Search results expired"
   - Show cached query params if available
   - Offer: "Run search again with same parameters"
```

**Key Interactions**:
- View query image (thumbnail + full resolution link)
- Scroll through match results
- Filter by similarity score
- Adjust page size
- Navigate pagination
- Click match to view details
- Open source link in new tab
- View bbox overlay on hover

**Expected Outcome**: Developer can view historical search results and understand what query produced which matches.

---

### Journey B: By userId â†’ Latest Search

**Scenario**: Developer wants to see the most recent search performed by a specific user/tenant.

**Flow**:
```
1. Developer opens URL: /search?userId={userId}
   â†“
2. Page loads â†’ Shows loading state
   â†“
3. API call: GET /api/v1/search/latest?userId={userId}
   â†“
4. [Success Path - Has recent search]
   - Redirect to: /search?id={searchId}
   - (Follow Journey A)
   
   [Success Path - Alternative: Show latest search inline]
   - Display latest search results directly
   - Show timestamp: "Latest search: 2 minutes ago"
   - Show query image and results
   
   [Error Path - No searches found]
   - Show empty state: "No searches found for user {userId}"
   - Offer: "Upload new image to search"
   
   [Error Path - userId invalid]
   - Show error state: "Invalid user ID"
   - Offer: "Return to home"
```

**Key Interactions**:
- Same as Journey A
- Additional: View search history (if available)
- Switch between recent searches

**Expected Outcome**: Developer can quickly access the most recent search for debugging/testing without knowing the searchId.

---

### Journey C: By uploadId â†’ Search for that Upload

**Scenario**: Developer has an `uploadId` from image upload logs and wants to see what face search results it produced.

**Flow**:
```
1. Developer opens URL: /search?uploadId={uploadId}
   â†“
2. Page loads â†’ Shows loading state
   â†“
3. API call: GET /api/v1/uploads/{uploadId}/search
   â†“
4. [Success Path - Upload has search results]
   - Display query image (from upload)
   - Display search results
   - Show metadata: upload timestamp, file name, size
   
   [Success Path - Upload exists, no search yet]
   - Show: "Upload found, but no search performed yet"
   - Offer: "Trigger search now" button
   - On click: Initiate search â†’ redirect to /search?id={newSearchId}
   
   [Error Path - uploadId not found]
   - Show error state: "Upload not found"
   - Possible reason: "Upload may have expired or been deleted"
   - Offer: "Upload new image"
   
   [Error Path - uploadId expired]
   - Show error state: "Upload expired"
   - Show expiry timestamp if available
   - Offer: "Upload same image again"
```

**Key Interactions**:
- View upload metadata (file name, size, timestamp)
- Same as Journey A for search results
- Trigger search if not performed yet
- Re-upload if expired

**Expected Outcome**: Developer can trace an uploadId to its search results and understand the complete flow from upload â†’ search â†’ results.

## 4. UI Components Specification

### 4.1 Query Panel
- **Location**: Top of page, full width
- **Height**: 200-250px (fixed)
- **Contents**:
  - Query image (left): 150x150px thumbnail
  - Query metadata (right): Upload time, tenant, parameters
  - View full resolution link
- **Behavior**:
  - Always visible (sticky on scroll, optional)
  - Image loads with placeholder/skeleton

### 4.2 Match Grid
- **Layout**: CSS Grid
- **Columns**: 5 per row (desktop), 3 (tablet), 1 (mobile)
- **Gap**: 20px
- **Card Size**: 150x150px thumbnail + 80px metadata = 230px height
- **Behavior**:
  - Virtual scrolling (render only visible cards)
  - Lazy load images (Intersection Observer)
  - Hover shows bbox overlay

### 4.3 Match List
- **Layout**: Vertical list, full width
- **Row Height**: 120px (thumbnail + metadata side-by-side)
- **Columns**: Thumbnail (100px) | Metadata (flexible) | Actions (120px)
- **Behavior**:
  - More metadata visible than grid view
  - Expandable rows for full details (optional)

### 4.4 Filters
- **Min Score Slider**:
  - Range: 0.00 - 1.00
  - Step: 0.01
  - Shows current value
  - Debounced (300ms)
- **Site Dropdown**:
  - Multi-select or single-select
  - Show result count per site
  - "All Sites" option
- **Date Range** (optional):
  - Two date pickers (from/to)
  - Preset ranges: Today, Last 7 days, Last 30 days

### 4.5 Pagination
- **Page Size Options**: 10, 25, 50, 100
- **Navigation**: Prev/Next buttons + page numbers
- **Display**: "Showing X-Y of Z results"
- **Behavior**:
  - Scroll to top on page change
  - Preserve filters across pages

### 4.6 BBox Overlay
- **Trigger**: Hover over match card
- **Display**: Dashed border overlay on thumbnail
- **Tooltip**: Show bbox coordinates `[x, y, w, h]`
- **Color**: Semi-transparent color (rgba)

### 4.7 Actions
- **View Source**: Opens source URL in new tab
- **Copy ID**: Copies face_id to clipboard
- **View Details** (optional): Expands card with full metadata

### 4.8 Debug Panel
- **Location**: Bottom of page (after results)
- **State**: Collapsed by default
- **Toggle**: Button to expand/collapse
- **Contents**: JSON response, timing, parameters
- **Actions**: Copy JSON, Download JSON

## 5. Responsive Behavior

### Desktop (> 1024px)
- 5 columns grid
- Sidebar filters (left)
- Full pagination controls

### Tablet (768px - 1024px)
- 3 columns grid
- Filters move to top (collapsible)
- Simplified pagination

### Mobile (< 768px)
- 1 column grid (or list view forced)
- Filters in drawer/modal
- Minimal pagination (prev/next only)

## 6. Interaction Details

### 6.1 Filter Application
1. User adjusts filter (slider, dropdown, etc.)
2. Debounce 300ms (wait for user to finish)
3. Apply filter client-side (re-filter existing results)
4. Update result count
5. Reset to page 1

### 6.2 View Source Action
1. User clicks "View Source" on match card
2. Open source URL in new tab: `window.open(url, '_blank')`
3. Handle expired URLs:
   - Catch error if URL fails to load
   - Show toast: "Source URL may have expired"

### 6.3 Copy ID Action
1. User clicks "Copy ID"
2. Copy face_id to clipboard: `navigator.clipboard.writeText(id)`
3. Show toast: "Face ID copied: face-123"

### 6.4 BBox Overlay
1. User hovers over match card
2. Show dashed overlay on thumbnail
3. Display tooltip with bbox coordinates
4. Hide on mouse leave

### 6.5 Pagination
1. User clicks page number or prev/next
2. Fetch new page of results (or re-slice client-side if all loaded)
3. Scroll to top of results section
4. Update URL query param: `?page=2`
